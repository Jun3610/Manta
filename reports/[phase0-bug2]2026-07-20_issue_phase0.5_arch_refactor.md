# [phase0-bug2] Phase 0.5 아키텍처 리팩토링 — 캘린더 도메인 로직 Python 분리

## 📖 개요
Phase 0 실사용 테스트에서 발견된 아키텍처 문제 4건을 국소 리팩토링으로 수정.
**핵심 원칙**: 캘린더 필터링·요일 계산·범위 계산은 Python이 처리, LLM은 자연어 변환만 담당 (SPEC 2.4절 확장).

---

## 📋 이슈 목록

### 1. 근무 조회 전용 경량 도구 신설 (`get_work_schedule`)
- **문제**: `get_events_by_range` / `search_events`는 92개 전체 일정을 반환 → LLM이 직접 필터링·요일 계산을 시도하여 개인 일정 혼합·요일 오류 발생.
- **해결**: `tools/calendar_tools.py`에 `get_work_schedule(range_str)` 도구 신설.
  - Python이 직접: `resolve_date_string()`으로 날짜 범위 변환 / "출근" 키워드 필터링 / `datetime.weekday()`로 요일 계산 후 JSON에 첨부.
  - LLM에는 필터·요일이 완성된 결과만 전달 → 재추측 불가 구조.
  - `core/agent.py` 시스템 프롬프트에 "근무 관련 질문에는 반드시 `get_work_schedule`만 사용, `get_events_by_range`로 대체 금지" 명시.
- **효과**: 필터링 정확도 ↑, 요일 정확도 ↑, 응답 속도 ↑ (걸러진 데이터만 LLM 전달).
- **수정 파일**: `tools/calendar_tools.py`, `core/agent.py`

---

### 2. 의도치 않은 `delete` 도구 재실행 조사 (안전 이슈 — 원인 미확정)
- **증상**: "다음주 돈 관련된거있음?" 조회 질문에 봇이 "휴식 일정 삭제 완료"라고 응답. 삭제와 무관한 질문이었으나 `delete` 도구 호출 의심.
- **조사 결과**:
  - `core/agent.py`의 tool-call 로그에서 해당 세션의 `delete` 재호출 여부를 직접 확인하지 못함 (로그 미보존, 재현 불가).
  - **유력 가설**: `SQLChatMessageHistory`의 `AttributeError: 'dict' object has no attribute 'type'` 오류로 히스토리 메시지가 `dict`로 깨지고, LLM이 직전 턴의 `tool_call` 메시지를 현재 요청의 맥락으로 오인했을 가능성.
  - `FileChatMessageHistory`로 교체(기존 버그리포트 #10) 후 히스토리 직렬화 문제 해소됨 → 재발 가능성 낮아짐.
- **현 조치**:
  - 시스템 프롬프트에 "사용자가 '삭제', '지워줘'를 명확히 명시한 경우에만 `delete` 도구 호출" 규칙 추가.
- **⚠️ 미해결**: 실제 재호출 여부 확인 불가. **안전 이슈이므로 추측으로 닫지 않음.** 다음 테스트에서 재현 시 로그 첨부 후 원인 확정 예정. 현재 상태: **Open**.
- **수정 파일**: `core/agent.py` (시스템 프롬프트)

---

### 3. 일정 추가 시 "확인 후 실행" 규칙 명확화
- **문제**: 시간이 불명확한 "내일 휴식 추가해줘" 요청에 기본값(09:00)으로 즉시 일정 추가. 사용자가 정정하려 해도 `add`를 반복 호출해 중복 일정 생성.
- **해결**: 시스템 프롬프트에 두 규칙 추가:
  - "시간이 불명확하면 도구 호출 전 먼저 확인 ('몇 시에 추가할까요?')"
  - "방금 만든 일정을 정정하는 것으로 판단되면 `add` 대신 `modify_apple_calendar_event` 사용"
- **수정 파일**: `core/agent.py` (시스템 프롬프트)

---

### 4. 과거 작업 이력 현재 응답에 재포함 — "물어본 것만 답하기"
- **문제**: 조회 질문에 이전 삭제/추가 완료 이력을 섞어서 답변하거나, 요청하지 않은 과거 작업 요약을 스스로 출력.
- **해결**: 시스템 프롬프트에 명시:
  - "과거 도구 실행 이력(삭제 완료, 추가 완료 등)을 현재 응답에 임의로 재포함하지 마라."
  - "지금 질문에 대한 답만 하라."
- **수정 파일**: `core/agent.py` (시스템 프롬프트)
