Phase 0: 캘린더 다중 업데이트 아키텍처 및 안전망(HITL/Rollback/Memory) 구축

## 📖 개요
기존 단일 Agent가 처리하기 어려웠던 복잡한 캘린더 다중 업데이트 로직을 LangGraph 기반으로 분리하고, 봇이 임의로 캘린더를 훼손하는 것을 방지하기 위해 사용자 승인(HITL) 및 롤백, 장기 메모리 시스템을 구축합니다.

## ✨ 구현 기능 목록
1. **LangGraph 기반 다단계 워크플로우 분리 (`calendar_bulk_update.py`)**
   - 기존 `AgentExecutor`의 단일 루프의 한계를 극복하기 위해 State Machine 도입.
   - **처리 흐름**: 날짜/조건 파싱 ➡️ 대상 이벤트 검색 ➡️ 변경 프리뷰 생성 ➡️ 사용자 승인 대기 ➡️ 승인 시 스냅샷 저장 및 실제 조작 ➡️ 패턴 학습.
2. **Human-in-the-loop (HITL) 사용자 승인 노드 (`approval.py`)**
   - 캘린더 데이터를 실제로 조작하기 전, 디스코드에 변경 전/후 프리뷰 발송.
   - 👍(승인) 또는 👎(거절) 리액션을 최대 5분간 대기하며, 명시적 승인이 있을 때만 진행.
3. **안전한 스냅샷 기반 롤백 시스템 (`rollback_store.py`)**
   - 일정 변경 직전 대상 이벤트들의 원본 상태를 로컬 디스크(JSON)에 스냅샷으로 백업.
   - "롤백해줘" 요청 시 가장 최근 스냅샷을 읽어 애플 캘린더의 기존 일정들을 원래대로 100% 복구.
4. **장기 메모리(학습) 시스템 연동 (`memory_service.py`)**
   - 일괄 수정 시 사용자가 묘사한 패턴(예: "OP 출근은 6시~15시")을 LLM이 자동 추출.
   - 추출된 선호도를 SQLite (`user_memory.db`)에 영구 저장하여 이후 봇의 맥락으로 활용.

## 🛠 작업 과정 (Work Process)
1. 인프라 준비: `data/user_memory.db`, 스냅샷 저장 디렉토리 등 로컬 스토리지 환경 구성.
2. Graph Node 모듈화: 각 단계별 로직(검색, 프리뷰, 승인, 실행, 롤백)을 독립된 파이썬 모듈로 개발.
3. ChatCog 연동: 디스코드 `on_message` 이벤트에서 복잡한 명령어를 인지하고 LangGraph를 호출하도록 라우팅 처리.
4. 테스트: "이번 주말 출근 전부 OP로 바꿔" 등의 명령어로 HITL 및 롤백 워크플로우 정상 동작 확인.

## 🎯 목표 및 승인 기준 (Acceptance Criteria)
- [x] 봇이 다량의 일정을 변경할 때 반드시 프리뷰를 띄우고 승인 리액션을 대기해야 한다.
- [x] 실수로 승인했더라도 "롤백해" 한 마디로 모든 변경 사항이 원상 복구되어야 한다.
- [x] 사용자의 고정적인 일정 패턴을 봇이 기억하여 다음 요청 시 활용할 수 있어야 한다.
